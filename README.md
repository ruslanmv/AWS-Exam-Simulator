## Why this exists

Studying for AWS certifications by reading docs and flipping flashcards doesn't work for most people. You forget what you "learned" within days. The research is clear: **active recall + spaced repetition + immediate feedback** is what actually moves knowledge into long-term memory.

This project started as a simple exam simulator -- pick an exam, answer questions, see your score. That's useful, but it's not enough. A static quiz can tell you *what* you got wrong; it can't teach you *why* or make sure you understood the concept before moving on.

The upgraded version adds an **AI tutor agent** that sits on top of the exam engine. It doesn't just grade your answers. When you get something wrong, it:

1. Explains the underlying AWS concept step by step
2. Points out the specific misconception that led you astray
3. Asks you a follow-up question to verify you actually understood
4. If you're still stuck, re-explains with a simpler analogy and asks you to restate the rule in your own words

It only moves on when you demonstrate understanding. That's what "teach until learned" means.

The tutor also tracks your weak areas across 17 AWS domains and adaptively selects questions that target where you need the most practice. Over time, it builds a profile of your knowledge gaps and fills them systematically.

## Architecture

```
                        +---------------------+
                        |    React Frontend   |
                        |  (Vite + Tailwind)  |
                        +----------+----------+
                                   |
            +----------------------+----------------------+
            |                                             |
  +---------v----------+                      +-----------v-----------+
  |   Legacy Gradio    |                      |   A2A Tutor Agent     |
  |   Backend (app.py) |                      |   (agent_runtime/)    |
  +--------------------+                      +-----------+-----------+
                                                          |
                                              +-----------v-----------+
                                              |   MCP Tool Server     |
                                              |  (mcp_server/)        |
                                              |                       |
                                              |  exam_list            |
                                              |  exam_start_session   |
                                              |  exam_next_question   |
                                              |  exam_submit_answer   |
                                              |  exam_get_explanation |
                                              |  session_get_status   |
                                              +-----------+-----------+
                                                          |
                                              +-----------v-----------+
                                              |  questions/*.json     |
                                              |  (20 exam banks)      |
                                              +-----------------------+
```

**Capability layer (MCP Server):** Deterministic tools that load question banks, manage sessions, grade answers, track per-tag accuracy, and provide remediation hints. No LLM needed -- pure logic.

**Behavior layer (A2A Agent):** An AI tutor that orchestrates the learning loop. It calls MCP tools for data and uses its own judgment to teach, adapt, and decide when to move on.

**Presentation layer (React + Gradio):** Two frontends -- the original React exam simulator UI and the legacy Gradio app -- both consuming the same question banks.

## Quick Start

### Option 1: AI Tutor (Interactive CLI)

The fastest way to start studying. No LLM API key needed -- the tutor runs in interactive mode with the exam engine.

```bash
# Install
make install-python

# Launch the tutor
make agent-cli
```

You'll see a list of available exams, pick one, and the tutor starts asking questions adaptively.

### Option 2: A2A HTTP Server

Run the tutor as an HTTP endpoint for integration with other systems (Context Forge, custom UIs, Slack bots, etc.).

```bash
make agent-server
# Server starts on http://localhost:8080
# Endpoints: /a2a/invoke, /a2a/health, /a2a/card
```

### Option 3: React Frontend

The original exam simulator UI with timer, question navigation, flagging, and instant feedback.

```bash
make install
make start
# Opens http://localhost:3000
```

### Option 4: Docker

```bash
docker compose up
# MCP Server: port 9101
# A2A Agent:  port 8080
```

## AI Tutor

The tutor agent operates in three modes:

| Mode | Behavior |
|------|----------|
| **Learning** | Adaptive question selection. Teaches on every wrong answer. Doesn't move on until you understand. Tracks 17 domain tags to find your weak areas. |
| **Practice** | All questions in sequence with immediate feedback. Good for review. |
| **Exam** | Timed. No feedback until the end. Simulates real exam conditions. |

### How the teach-until-learned loop works

```
1. Agent calls exam_next_question (biased toward your weak tags)
2. Presents question with options
3. You answer
4. Agent calls exam_submit_answer
5. If CORRECT:
   - Brief confirmation + one related concept
   - Move to next question
6. If INCORRECT:
   - 3-6 step explanation of the AWS concept
   - Why your choice was wrong (common misconception)
   - Why the correct answer is right
   - Follow-up check question
   - If you're still stuck: simpler analogy, ask you to restate the rule
   - Only proceed when you demonstrate understanding
7. Every 5-10 questions: session_get_status check
   - Report accuracy and mastery level
   - Adjust focus toward weak domains
```

### Domain tags tracked

The auto-tagging engine classifies every question into one or more of 17 domains:

`iam` `vpc_networking` `s3_storage` `ec2_compute` `serverless` `containers` `databases` `monitoring_logging` `security_encryption` `high_availability` `cost_optimization` `devops_cicd` `ml_ai` `messaging_integration` `content_delivery` `analytics` `migration`

## MCP Server

The MCP server exposes six tools over the [Model Context Protocol](https://modelcontextprotocol.io/) standard. Any MCP-compatible client can consume them.

| Tool | Description |
|------|-------------|
| `exam_list` | List all exams with titles and question counts |
| `exam_start_session` | Start a new session (learning / practice / exam) |
| `exam_next_question` | Get next question (adaptive selection in learning mode) |
| `exam_submit_answer` | Submit answer, get correctness + explanation + remediation |
| `exam_get_explanation` | Fetch explanation and references for any question |
| `session_get_status` | Accuracy, weak/strong tags, mastery level, remaining questions |

### Running standalone

```bash
export AWS_EXAM_QUESTION_DIR=./questions
aws-exam-tools   # stdio transport
```

### Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `AWS_EXAM_QUESTION_DIR` | `./questions` | Path to `*.json` question banks |
| `AWS_EXAM_DB_PATH` | `./state/aws_exam.sqlite` | SQLite database for sessions |

## API Integration

The React frontend uses a service layer (`src/services/api.js`) for backend communication. Configure your backend URL with the `VITE_API_BASE_URL` environment variable.

### REST API Endpoints

#### Exam APIs
- `GET /exams` -- List available exams
- `GET /exams/:examId` -- Get exam data with questions
- `POST /exams/submit` -- Submit exam results
- `GET /exams/history` -- User's exam history
- `GET /exams/results/:attemptId` -- Detailed results

#### Admin APIs
- `POST /admin/login` -- Admin authentication
- `POST /admin/logout` -- Admin logout
- `GET /admin/logs` -- All exam attempts (with filters)
- `GET /admin/stats` -- Dashboard statistics

#### User APIs
- `POST /users/register` -- User registration
- `POST /users/login` -- User authentication
- `GET /users/profile` -- Get user profile

### A2A Agent Endpoints

When running the tutor agent as an HTTP server (`make agent-server`):

- `POST /a2a/invoke` -- Send a message to the tutor agent
- `GET /a2a/health` -- Health check
- `GET /a2a/card` -- Agent card (A2A discovery metadata)

## Development Notes

### Mock data

If the backend API is unavailable, the React frontend falls back to mock data for development. This allows frontend work without a running backend.

### Authentication

- Student routes are accessible without authentication (uses mock user)
- Admin routes require authentication via `/admin` login
- Protected routes redirect to login if not authenticated

### State management

The React app uses Context API:
- **ExamContext** -- Exam state, timer, answers, navigation
- **AuthContext** -- User authentication and session

### Question bank formats

The MCP server auto-detects two JSON formats:

**Format A** (SAA-C03 style):
```json
{"question": "...", "options": ["A. ...", "B. ..."], "correct": "A. ...",
 "explanation": "...", "references": "https://..."}
```

**Format B** (CLF-C02 style):
```json
{"question": "...", "options": ["option1", "option2"], "correct": "option1"}
```
